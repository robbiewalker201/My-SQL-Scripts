Relational databases:
Designed to only store the minimum amount of data that is needed in that tabl
No unecessary duplicated data between relational tables
We can do this by using primary keys to relate to other tables that stores the additional data

SQL Statements:

ollama request:

In Python you can do a 'Type annotation' or type hint

It is a way of returning an object of a class as a variable.

It never actually calls the class, so never runs __init__
It just annotates the object with that class name

The annotation is stored as metadata, nothing actually happens to it
If you want to check that the object is of the right class as annotated, you can do this manually

Cursors:
Cursors act as readers for the user to read over rows in a database table.
The cursor - executes SQL, fetches rows, tracks positions
Cursors are explicity open or closed

Cursors solve three main problems
- stream data so you dont have to load everything at once
- track where you are in a table
- allow for a single interface across a whole database

ORM - Object Relational Mapping
ORMs hide cursors

All database operations must go through a connection, so a connection is needed for active sessions
This means the database connection must also be closed at the end of your session
When closing the connection the following things are done 
- Database is locked
- Releases File Handlers
- Releases memory used by SQLLite

If you do not close the connection then the database could be left locked for future code to fail
